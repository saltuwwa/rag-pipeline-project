# GraphRAG — Бонусное задание

Knowledge Graph на основе отчётов КТЖ и Матен Петролеум в Neo4j.

## Где посмотреть граф?

**Neo4j Browser** — веб-интерфейс для работы с базой:

1. Запустите Neo4j (Docker) и загрузите граф из ноутбука
2. Откройте в браузере: **http://localhost:7474**
3. Войдите: логин `neo4j`, пароль `password123`
4. Выполните Cypher-запросы, например:
   ```cypher
   MATCH (n)-[r]->(m) RETURN n, r, m LIMIT 100
   ```
   или
   ```cypher
   MATCH (c:Company)-[:HAS_PROJECT]->(p:Project) RETURN c.name, p.name
   ```

## Трудности и улучшения

### Проблема 1: LLM возвращал 0 сущностей и 0 связей

**Причина:** В `ChatPromptTemplate` фигурные скобки `{` и `}` из примера JSON интерпретировались как placeholders при вызове `prompt.format()`, из-за чего возникало исключение и возвращались пустые списки.

**Решение:** Отказ от ChatPromptTemplate, переход на f-string. В промпте для literal-скобок используется экранирование `{{` и `}}` в f-string.

### Проблема 2: Множественные дубликаты сущностей

**Причина:** Одна и та же сущность (например, «Матен Петролеум» и «МАТЕН ПЕТРОЛЕУМ») извлекалась из разных чанков и добавлялась много раз.

**Решение:** Дедупликация по нормализованному имени (`(name.lower(), type)`), объединение `source` из нескольких документов, предпочтение варианта в Title Case вместо ALL CAPS.

### Проблема 3: Пустые связи при работе через fallback

**Причина:** Fallback находил только сущности по регулярным выражениям, без автоматического создания связей между ними.

**Решение:** 
- Добавлены эвристические связи: Company → Project (HAS_PROJECT), Company → Location (LOCATED_IN)
- При отсутствии подходящих типов — цепочка RELATED_TO для связности графа

### Проблема 4: Мало сущностей при работе только fallback

**Причина:** Изначально fallback содержал всего ~7 паттернов.

**Решение:** Расширенный набор паттернов: компании (ҚТЖ, Кожан, КазМунайГаз, Chevron, ExxonMobil, PwC, GRI и др.), проекты (Кара-Арна, Матин, Тенгиз, КТК), метрики (ESG, EBITDA, NOPAT), локации (Атырау, области). Увеличено число обрабатываемых чанков до 25 на документ.

### Итог

- **LLM:** ~270 сущностей, ~181 связь (типы HAS_PROJECT, HAS_METRIC, HAS_LOCATION и др.)
- **Fallback и эвристики** используются как резерв при пустом ответе LLM
- Граф можно просматривать в Neo4j Browser и использовать для Cypher-запросов по типовым вопросам
